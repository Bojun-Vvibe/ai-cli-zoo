# grex

> **A command-line tool (and Rust library with Python
> bindings) for generating regular expressions from
> user-provided test cases** ÔÇö feed it a list of strings and
> it emits the smallest regex that matches all of them, with
> optional Unicode-class collapsing, repetition detection, and
> verbose-mode output. Pinned to **v1.4.6**
> ([LICENSE](https://github.com/pemistahl/grex/blob/main/LICENSE),
> Apache-2.0).

Source: <https://github.com/pemistahl/grex>

## TL;DR

`grex` is the inverse of "test this regex against these
strings" ÔÇö you give it the strings, it gives you the regex.
`grex foo bar baz` emits `^(?:ba[rz]|foo)$`. With `-d` it
detects digit / non-digit / word-character classes
(`grex 'a1' 'b2' -d` Ôćĺ `^[a-b]\d$`). With `-r` it detects
repetition (`grex 'aaa' 'aaaa' -r` Ôćĺ `^a{3,4}$`). Useful when
you have 50 example log lines and need a regex that catches
the family without writing it by hand.

## Install

```bash
# Homebrew (macOS / Linux)
brew install grex

# Cargo (any OS with a Rust toolchain)
cargo install grex

# Pip (Python bindings)
pip install grex

# verify
grex --version    # grex 1.4.6

# basic: literal alternation
grex foo bar baz                              # ^(?:ba[rz]|foo)$

# detect digit class + repetition
grex -d -r 'item1' 'item2' 'item22'           # ^item\d{1,2}$

# read inputs from a file (one per line)
grex -f log-samples.txt
```

## Niche It Fills

**Regex synthesis from examples.** Most of the time you don't
*want* to write a regex ÔÇö you want to describe a set of
strings to match, and the regex is plumbing. `grex` flips the
loop: paste your sample strings, get a regex, tweak it. For
log scraping, sed-substitution prep, and grep filters across
heterogeneous corpora, this collapses 30 minutes of trial-and-
error into 30 seconds.

## Why it pairs with coding agents

Agents are middling at writing regexes from prose
descriptions ÔÇö they often produce something that matches the
example in the prompt but fails on the next 20 inputs.
`grex` is a deterministic ground-truth generator: paste 10
real samples, get a regex that demonstrably matches all 10,
and the agent can refine from a working baseline instead of
guessing from prose. Especially useful inside [`aichat`](../aichat/)
/ [`shell-gpt`](../shell-gpt/) shell pipelines: `grex
$(grep ERROR app.log | head -20)` produces a starter regex the
agent can then polish.

## Vs Already Cataloged

- **Vs hand-written regex + `ripgrep`:** `ripgrep` *uses* a
  regex you already have; `grex` *makes* the regex from
  examples. Pair them.
- **Vs an LLM regex prompt:** LLMs hallucinate regex syntax
  edge cases (PCRE vs RE2, lookbehind support, Unicode
  classes); `grex` is deterministic and emits valid regex
  every time.

## Caveats

- **Output is verbose by default.** It anchors with `^...$`
  and uses non-capturing groups. Add `--no-start-anchor` /
  `--no-end-anchor` if you don't want anchoring; pipe through
  a formatter if you want it pretty.
- **Worst-case complexity is exponential** in input count for
  the repetition detector (`-r`). For >1000 sample strings,
  drop `-r` or pre-deduplicate.
- **Not all regex flavours.** Output is RE2-compatible
  (`ripgrep`, Go, Rust regex crate); some PCRE features
  (lookbehind, named-group backrefs) are not generated by
  design.
