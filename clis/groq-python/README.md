# groq-python

> Snapshot date: 2026-04. Upstream: <https://github.com/groq/groq-python>
> Binary name: none ‚Äî Python SDK; usable as `python -m groq` patterns
> and via the cross-CLI `--provider groq` flag in `tgpt`, `mods`,
> `aichat`, `llm`, and `shell-gpt`.

`groq-python` is the **fast-inference provider hook** of the catalog.
Groq's LPU-backed API serves a fixed list of open-weights models
(`llama-3.3-70b-versatile`, `llama-3.1-8b-instant`,
`mixtral-8x7b-32768`, `gemma2-9b-it`, plus Whisper for audio) at
latencies that are noticeably below GPU-hosted alternatives ‚Äî
hundreds of tokens per second on the 70B-class models is the
distinguishing fact.

This entry is the **canonical Python SDK** for that API. It is
OpenAI-SDK-shaped on purpose ‚Äî `from groq import Groq; client =
Groq(); client.chat.completions.create(...)` ‚Äî so any tool that
already knows how to talk to OpenAI can be pointed at Groq with a
base-URL change. That compatibility is why nearly every other CLI in
this catalog has a `--provider groq` or `model: groq/...` flag: they
re-export this SDK or hit the same HTTP shape.

The SDK itself is a thin, autogenerated client (Stainless-generated,
typed, with sync + async surfaces and streaming support). The reason
to install it directly rather than going through one of the wrappers
is when you are writing your own scripts, a `litellm`-free agent
loop, or a one-off benchmark where you want exact control over the
request shape and tokens-per-second numbers.

## 3. Install

```bash
pip install -U groq
export GROQ_API_KEY=gsk_...                        # from https://console.groq.com/keys
python - <<'PY'
from groq import Groq
client = Groq()
resp = client.chat.completions.create(
    model="llama-3.3-70b-versatile",
    messages=[{"role": "user", "content": "explain LPUs in one sentence"}],
)
print(resp.choices[0].message.content)
PY
```

Or, for cross-CLI use:

```bash
tgpt --provider groq --model llama-3.3-70b-versatile "summarize this log" < error.log
```

## 4. When to choose

- You want **the fastest hosted Llama-3.x inference available** for
  interactive CLI use ‚Äî Groq's TPS on 70B beats GPU-hosted offerings
  by a wide margin.
- You are building a **custom agent loop in Python** and want a
  typed, OpenAI-shaped SDK without dragging in `langchain` or
  `litellm` for one provider.
- You are running **whisper-large-v3 transcription at scale** ‚Äî
  Groq's audio endpoint is one of the cheaper hosted options per
  minute, and the SDK's `client.audio.transcriptions.create()`
  surface is identical to OpenAI's.
- Avoid if you need **closed-weights frontier models** (Claude
  Opus/Sonnet, GPT-4-class, Gemini Ultra) ‚Äî Groq does not serve
  those. Use the matching provider SDK.
- Avoid if you need **on-prem** ‚Äî Groq is hosted only. For local
  Llama-3.3-70B, see [`vllm`](../vllm/), [`sglang`](../sglang/),
  [`lmdeploy`](../lmdeploy/), or [`llama.cpp`](../llama.cpp/).
